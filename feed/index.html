OpenClaw for Synology

Unofficial Synology DSM package that installs OpenClaw via Container Manager. Install, upgrade, and uninstall from Package Center like any other DSM package.

Add the Package Source

  1. On DSM, open Package Center → Settings → General, set Trust Level to Any publisher.
  2. Open Package Center → Settings → Package Sources → Add.
  3. Name: warting. Location:

https://warting.github.io/synology-openclaw/

  1. OK. Switch to the Community tab. OpenClaw should appear.
  2. Click Install. The wizard will ask for the web UI port (default 18789) and, optionally, an Ollama host. Leave Ollama blank if you don't use it — you can configure any provider (Anthropic, OpenAI, Ollama, LM Studio, …) from OpenClaw's own onboarding UI after install.

What gets installed

  • A Docker container from ghcr.io/openclaw/openclaw managed via docker compose.
  • Persistent data under /volume1/docker/openclaw/ (config, workspace).
  • A desktop icon on DSM that opens http://<your-nas>:18789/.

Safety defaults

  • No /var/run/docker.sock is mounted. OpenClaw's built-in Docker sandbox is left off on purpose — mounting the socket would be equivalent to giving the container root on your NAS, which defeats the purpose of containment.
  • No network isolation is configured by default. If you want stricter isolation (separate VLAN), do that in your router/switch.
