OpenClaw for Synology

Run OpenClaw on a Synology NAS via Container Manager. See github.com/warting/synology-openclaw for the compose file and install instructions.

Install via Container Manager

  1. DSM → Container Manager → Project → Create.
  2. Project name: openclaw. Path: docker/openclaw in your docker shared folder.
  3. Source: Create docker-compose.yml. Paste the compose.yml from the GitHub repo linked above.
  4. Next → Done. Visit http://<your-nas-ip>:18789/.

SPK package (future)

A Synology Package Center .spk is in progress. DSM 7.2's unsigned-package validator currently refuses third-party packages that don't run as root and also refuses anything that does, so the SPK path is blocked until code signing is sorted out. Source lives in the repo above.

What gets installed

  • A Docker container from ghcr.io/openclaw/openclaw managed via docker compose.
  • Persistent data under /volume1/docker/openclaw/ (config, workspace).
  • A desktop icon on DSM that opens http://<your-nas>:18789/.

Safety defaults

  • No /var/run/docker.sock is mounted. OpenClaw's built-in Docker sandbox is left off on purpose — mounting the socket would be equivalent to giving the container root on your NAS, which defeats the purpose of containment.
  • No network isolation is configured by default. If you want stricter isolation (separate VLAN), do that in your router/switch.
